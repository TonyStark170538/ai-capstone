# AI Capstone Project

An AI engineering capstone that explores modern JavaScript development with AI-assisted workflows.

This repository demonstrates how to build software with Node.js, Git, GitHub, and Cursor AI while keeping the project organized, documented, and easy to maintain.

## Technology Stack

- **Node.js** — runtime environment
- **JavaScript** — application language
- **Cursor AI** — AI-assisted development
- **Git & GitHub** — version control and collaboration

## Purpose

This project showcases AI-assisted development practices, including:

- Planning and implementing features with an AI coding assistant
- Maintaining clear project documentation
- Following consistent coding and Git conventions
- Building a clean, professional repository structure

## Features

- AI-assisted coding workflow with Cursor
- Node.js and JavaScript application foundation
- Version control with Git and GitHub
- Documented project conventions in `CLAUDE.md`
- MIT licensed open-source setup

## Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) (LTS recommended)
- [Git](https://git-scm.com/)
- [Cursor](https://cursor.com/) (optional, for AI-assisted development)

### Installation

Clone the repository:

```bash
git clone https://github.com/TonyStark170538/ai-capstone.git
cd ai-capstone
```

Install dependencies:

```bash
npm install
```

## Project Structure

```text
ai-capstone/
├── README.md      # Project overview and setup
├── LICENSE        # MIT License
├── .gitignore     # Ignored files and folders
└── CLAUDE.md      # Coding and Git conventions for AI assistants
```

## Development

When working in this repository:

1. Read `CLAUDE.md` for coding and commit conventions
2. Keep changes focused and well documented
3. Use [Conventional Commits](https://www.conventionalcommits.org/) for commit messages

Example commit messages:

```text
feat: add user authentication
fix: handle missing environment variables
docs: update README setup instructions
```
